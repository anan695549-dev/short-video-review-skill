# 短视频复盘

把发布后的真实数据接回内容生产链。目标不是解释所有数字，而是找到当前最大的瓶颈，并生成下一轮可验证的动作。

## 输入处理

接受以下任一输入：

- 平台后台截图：读取可见字段；看不清或平台未提供的字段留空，不猜。
- CSV：按列名映射字段；保留原始文件，不覆盖。
- XLSX：调用电子表格能力读取和计算，不手工拆 XML。
- 手工数据：按 [review-data-template.csv](assets/review-data-template.csv) 的字段收集。
- 视频链接或文案：只能做内容诊断，不能冒充数据复盘。

需要平台字段解释或截图清单时，读取 [platform-metrics.md](references/platform-metrics.md)。

先确认：平台、统计窗口、视频数量、账号目标。目标只能从以下四项里选主目标：播放、互动、涨粉、私域/成交。用户未说明时，根据现有材料推断并标注假设，不用长问卷阻断工作。

## 样本闸门

| 样本 | 可以做 | 不可以做 |
|---|---|---|
| 1条 | 单条急诊、定位疑似漏斗问题、给下一条对照测试 | 宣称账号规律或最佳发布时间 |
| 2至4条 | 横向对比、提出暂定假设 | 宣称稳定模式 |
| 5至14条 | 计算账号基线、找明显优劣、安排一轮测试 | 做强因果判断 |
| 15至29条 | 按题材、Hook、形式、时长、发布时间找模式 | 忽略样本分布差异 |
| 30条以上 | 稳定模式分析、阶段策略调整 | 用历史相关性冒充因果 |

样本不足时照常给有边界的结果，不要只回复“数据不够”。

## 统一数据

每条视频尽量整理为：

```text
platform, content_id, publish_time, topic, format, hook_type,
duration_seconds, impressions, views, three_second_views,
average_watch_seconds, completion_rate, likes, comments,
shares, saves, profile_visits, follows, dms, leads
```

允许缺失字段。所有比例优先使用平台后台原值；需要计算时：

- 播放率 = 播放 / 曝光
- 3秒留存率 = 3秒观看 / 播放
- 平均观看占比 = 平均观看秒数 / 视频时长
- 互动率 = (点赞 + 评论 + 分享 + 收藏) / 播放
- 分享率、收藏率、评论率 = 对应动作 / 播放
- 主页访问率 = 主页访问 / 播放
- 关注转化率 = 新增关注 / 播放
- 线索转化率 = 私信或有效线索 / 播放

分母缺失时不计算。不同平台不要直接比较原始播放量；先在同平台、相近时间窗和相近内容形式内比较。

## 建立账号基线

有5条以上数据时，以当前窗口的中位数为主基线，平均值只作补充。爆款容易拉高平均值，中位数更适合判断常态。

至少计算：

- 播放或曝光中位数
- 3秒留存或平台最接近的前段留存
- 平均观看占比或完播率
- 分享率、收藏率、评论率
- 关注或线索转化率

比较作品时标注相对基线：`高于基线`、`接近基线`、`低于基线`。没有用户自己的基线时，不引用未经核验的行业平均值，也不设置绝对通过线。此时只定义配对实验和相对提升目标，例如“实验组比同批对照组高20%”；数据回收后再建立账号基线。

## 五段漏斗诊断

按顺序诊断，找到最早出现的明显下滑点。下游数据差可能只是上游流量不足，不要同时判五个问题。

### 1. 曝光

观察曝光、推荐量、搜索流量和发布时间。曝光低但后续留存、互动、转化高于账号基线时，优先判断为分发、封面标题、题材匹配或发布时间假设，不先重写正文。

### 2. 停留

观察封面点击、播放率、前1至3秒留存。曝光正常而前段留存显著低于自身基线时，优先诊断 Hook、第一画面、口播第一句或标题承诺不一致。

### 3. 留存

前段留存正常，但平均观看占比、完播或关键节点掉点明显时，诊断正文结构：铺垫过长、价值出现过晚、信息重复、转场断裂、案例不足或结尾拖沓。

### 4. 共鸣

观看指标正常，但分享、收藏、评论低时，诊断内容价值和情绪机制：观点不够具体、缺少可带走的信息、没有讨论空间，或平台表达方式不匹配。

### 5. 转化

观看和互动正常，但主页访问、关注、私信或线索低时，诊断账号承接、身份信号、CTA、资料承诺和主页内容，不把问题继续归咎于流量。

相对自身中位数低于约25%可作为“明显偏低”的排查信号，但不是通用行业阈值。样本小、发布时间不足24小时或存在投流时必须降级结论置信度。

## 模式分析

样本达到15条后，按以下维度分组；每组至少3条才给方向性判断：

1. 题材 / 内容支柱
2. Hook机制
3. 表达形式：口播、案例、清单、实测、观点、剧情、采访
4. 视频时长区间
5. 开场画面和字幕形式
6. 发布时间
7. 平台
8. CTA类型

分别寻找：

- 高播放、高留存、高转化的稳定组合
- 高播放、低转化的虚热内容
- 低播放、高转化的小而准内容
- 高收藏但低评论、或高评论但低关注的功能差异
- 连续出现而非单条偶然的模式

用“相关”“可能”“下一轮验证”表述，不使用“因为算法”“一定会火”等伪因果结论。

## 下一轮实验

每轮最多安排3个实验，每个实验只改变一个主变量：

```text
实验假设：
保持不变：题材 / 正文 / 时长 / 发布时间 / CTA
唯一变量：Hook / 第一画面 / 时长 / 结构 / CTA / 封面标题
对照组：
实验组：
核心指标：
观察窗口：
通过标准：相对账号当前基线提升多少
下一步：保留 / 再测 / 放弃
```

不要同一条视频同时更换题材、Hook、剪辑、时长和CTA，否则复盘无法知道什么起作用。

把用户能感知的一组联动元素视为一个实验变量，并写清边界：

- `封面标题组合`只包含封面主标题和副标题；首帧、口播第一句、正文不变。
- `开场兑现`只包含首帧和口播第一句；封面标题、正文、时长不变。
- `Hook机制`只改开头表达机制；同一题材成对测试，不能用不同题材分别代表不同Hook。
- 没有账号基线时，通过标准只能写相对对照组的提升幅度，不能写来源不明的绝对百分比。

## 失败与降级分支

- 如果截图字段看不清或分母缺失 -> 对应指标标记`未知`，列出需要补采的字段，不估算、不用相邻指标代替。
- 如果发布时间不足24小时或数据仍在明显增长 -> 只做低置信度急诊，给出复查时间，不定性账号规律。
- 如果样本不足5条且没有账号历史基线 -> 只提出疑点和配对实验，不使用`偏高`、`偏低`、`达标`等绝对判断。
- 如果实验同时改变两个以上主变量 -> 判定本轮不可归因；保留结果作观察，不把它沉淀为有效模式，下一轮重新做单变量对照。
- 如果跨平台只有播放中位数和线索总量 -> 只比较每条内容线索数和最终线索总量；不把中位播放当总播放计算线索率。
- 如果缺少投入工时、成本、线索质量或成交结果 -> 只给平台投入优先级，不给`70/30`等精确资源比例；先补齐`有效线索/条`、`有效线索/投入小时`、`成交/有效线索`。
- 如果平台指标口径不同，例如抖音完播与视频号完整观看 -> 分平台解释，不直接计算倍数或排名。

## 平台解释

- 抖音：优先检查前段停留、平均观看、完播、分享和关注；不要只看点赞。
- 小红书：同时检查封面点击、搜索/推荐来源、收藏、分享和主页访问；“有用”通常比泛点赞更重要。
- 视频号：重点看完整观看、转发、朋友推荐、关注和私域动作；不要只按抖音逻辑解释。
- X：重点看停留、观看、回复、转发、引用和主页访问；保留可以被讨论和反驳的空间。

详细字段映射见 [platform-metrics.md](references/platform-metrics.md)。

## 下游交接

只把已经定位的问题交给下游，不让下游重做整套复盘：

### 🔴 CHECKPOINT · 🛑 STOP

调用下游Skill批量改写、重剪或生成发布物之前，先向用户展示并确认以下四项：`首要问题`、`证据与置信度`、`唯一测试变量`、`不能改变的部分`。用户未确认时，只输出复盘和实验卡，不执行下游修改。

| 问题 | 交接 |
|---|---|
| 第一秒、第一句、Hook机制 | `dbs-hook` |
| 正文结构、口播节奏、结尾 | `pragmatic-ai-writer` 或 `dbs-script-flow` |
| 平台表达、合规、发布策略 | `traffic-compass` |
| 小红书封面和图文 | `xhs-visual-director`、`guizang-social-card-skill` |
| 剪辑节奏、包装、重剪 | `talking-head-recut`、`hyperframes` |
| 选题来源不足 | `business-ai-radar` |
| 有效模式需要沉淀成系列 | `dbs-content-system` |

每个交接任务必须写明“唯一测试变量”和“不能改变的部分”。

## 禁区

- 不用单条爆款定义账号规律。
- 不跨平台直接比较播放量。
- 不把相关性写成因果。
- 不为填满报告而编造缺失指标。
- 不建议一次改动多个变量。
- 不只给“多发、坚持、优化内容”等空泛建议。
- 不把点赞当作唯一成功指标。
- 不分析个人身份、隐私或与复盘无关的后台信息。
