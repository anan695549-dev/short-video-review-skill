# 短视频运营复盘 Skill

把抖音、小红书、视频号和 X 的真实发布数据接回内容生产流程，定位当前最早、最重要的漏斗问题，并生成下一轮可验证的单变量实验。

它不是爆款预测器，也不会拿一条视频定义整个账号。数据不足时会明确标注未知项、降低置信度，并告诉你下一步需要补什么数据。

## 能做什么

- 读取平台后台截图、CSV、XLSX 或手工数据。
- 复盘曝光、停留、留存、共鸣和转化五段漏斗。
- 用账号自己的中位数建立基线，不套用未经核验的行业均值。
- 支持单条急诊、5条以上阶段复盘、15条以上模式分析。
- 区分抖音、小红书、视频号和 X 的平台目标与指标口径。
- 输出下一轮单变量实验卡，避免一次修改多个因素后无法归因。
- 可生成适合沉淀到 Obsidian 的复盘卡片。

## 适用场景

- 为什么播放不低，但没有涨粉？
- 为什么小红书收藏高，主页访问和关注却很少？
- 最近10条视频最大的问题在哪里？
- 抖音和视频号应该优先投入哪一个？
- 下一条究竟先改 Hook、正文、封面还是 CTA？

## 安装

### 方法一：Skills CLI

仓库发布后可执行：

```bash
npx -y skills add anan695549-dev/short-video-review-skill -g --all
```

### 方法二：Codex 手工安装

下载仓库后，把整个目录放到：

```text
~/.codex/skills/short-video-review/
```

然后重启 Codex。

### 方法三：其他兼容 Runtime

把整个目录放入该工具支持的 Skills 目录，例如：

```text
~/.agents/skills/short-video-review/
```

不同 Runtime 的技能目录可能不同，请以其官方说明为准。

## 怎么调用

直接点名：

```text
调用 /short-video-review，复盘我最近10条抖音数据，主目标是涨粉。
```

也可以自然表达：

```text
这是我最近两周的视频号后台截图。帮我找出私域线索增长的首要瓶颈，设计下一轮测试。
```

```text
这条小红书发布18小时，曝光25000、观看1800、点赞110、收藏95。为什么表现一般？下一条只改一个变量。
```

```text
对比抖音和视频号最近15条数据。目标是私域线索，不要直接比较原始播放量。
```

## 最好提供哪些数据

最低输入：

- 平台
- 统计时间窗口
- 视频数量
- 当前主目标：播放、互动、涨粉、私域/成交，四选一

尽量补充：

- 曝光与播放
- 3秒留存、平均观看、完播率
- 点赞、评论、分享、收藏
- 主页访问、新增关注
- 私信、有效线索或成交

字段不完整也能运行。Skill 会把缺失字段标记为未知，不会编造数据。

可直接使用 [数据模板](assets/review-data-template.csv) 整理数据。

## 默认输出

1. 一句话判断：首要瓶颈、证据和置信度。
2. 数据可信度：样本量、缺失字段、统计窗口。
3. 五段漏斗诊断：曝光、停留、留存、共鸣、转化。
4. 做对了什么、应该停止什么。
5. 下一轮单变量实验卡。
6. 可选的下游任务单和 Obsidian 复盘卡。

## 样本边界

| 样本量 | 可以做 | 不应该做 |
|---|---|---|
| 1条 | 单条急诊、提出疑点 | 宣称账号规律 |
| 2至4条 | 横向比较、提出假设 | 宣称稳定模式 |
| 5至14条 | 建立账号基线、安排测试 | 做强因果判断 |
| 15至29条 | 分组寻找方向性模式 | 忽略样本分布差异 |
| 30条以上 | 阶段策略分析 | 把相关性写成因果 |

## 可选 Skill 交接

核心复盘能力不依赖其他 Skill。安装了以下 Skill 后，可以在用户确认首要问题和唯一测试变量之后继续交接：

- Hook：`dbs-hook`
- 口播结构：`pragmatic-ai-writer`、`dbs-script-flow`
- 平台策略：`traffic-compass`
- 小红书封面：`xhs-visual-director`、`guizang-social-card-skill`
- 剪辑：`talking-head-recut`、`hyperframes`
- 选题：`business-ai-radar`
- 内容系统：`dbs-content-system`

这些依赖全部是可选项；没有安装时，Skill 仍会输出完整复盘和实验任务单。

## 隐私说明

- Skill 本身不包含 API Key、账号密码、Cookie 或个人数据。
- 分析后台截图时，只读取复盘需要的指标，不处理无关个人身份信息。
- 是否把数据发送给模型，取决于你使用的 Runtime 和模型服务，请遵守对应平台的数据政策。
- 公开提交问题或案例前，请先遮盖账号名称、用户昵称、订单号和联系方式。
